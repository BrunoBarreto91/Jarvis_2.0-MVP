🔐 BLUEPRINT: AWS COGNITO AUTHENTICATIONEste documento define os requisitos técnicos para o provisionamento do sistema de identidade do Jarvis 2.0.1. Recursos Terraform (aws_cognito_*)O Agente deve adicionar ao arquivo main.tf os seguintes recursos:RecursoParâmetros Sugeridosaws_cognito_user_poolNome: jarvis-user-pool. Permitir login via email. Atributos obrigatórios: email.aws_cognito_user_pool_clientNome: jarvis-client. Explicit_auth_flows: USER_PASSWORD_AUTH, ALLOW_REFRESH_TOKEN_AUTH.aws_cognito_user_pool_domainDomínio: jarvis-auth-bunker (ou similar disponível).2. Outputs RequeridosApós o apply, o sistema deve expor:user_pool_iduser_pool_client_idregion (us-east-1)3. Segurança do BunkerMFA: Opcional nesta fase, mas o pool deve estar preparado para ativação futura.Auto-verificação: Configurar auto-verificação de e-mail para evitar intervenção manual no console.
